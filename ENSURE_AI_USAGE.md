# Ensure Exploit Generator is Using AI (Not Template)

## Quick Verification

Run this command to verify AI is configured and working:
```bash
python VERIFY_AI_USAGE.py
```

This will check:
1. ✓ Configuration file exists
2. ✓ API keys are set
3. ✓ AI providers available
4. ✓ AI generator initializes
5. ✓ ExploitTemplate has AI enabled
6. ✓ AI can actually generate code

## What Changed

The exploit generator was modified to:

1. **Force AI generation first**
   - Checks if AI is properly initialized
   - Logs provider name and client type
   - Validates before attempting generation

2. **Add detailed logging**
   - Shows which provider is being used
   - Logs character count of generated code
   - Tracks AI vs template-based generation
   - Full error stack traces on failure

3. **Verify AI initialization**
   - Checks `template.ai_gen` is not None
   - Logs provider: Mistral, OpenAI, Ollama, or Azure
   - Returns empty dict if AI fails (forcing fallback to prompts)

## How to Ensure AI is Used

### Step 1: Run Verification
```bash
python VERIFY_AI_USAGE.py
```

All checks should show ✓ PASS

### Step 2: Check Configuration
Verify `.hades_config.json` has your API key:
```json
{
  "mistral_api_key": "YOUR_KEY_HERE",
  "ai_provider": "mistral"
}
```

### Step 3: Monitor HadesAI Output
When you click "Generate Exploits", you should see in the console:
```
INFO: Available AI providers: ['mistral']
INFO: Using AI provider: mistral
INFO: AI Generator initialized with provider: mistral
INFO: Calling AI to generate Buffer Overflow
INFO: Mistral generated 2048 characters
✓ Buffer Overflow - AI-generated
```

### Step 4: Inspect Generated Code
Real AI-generated code will:
- Have detailed comments
- Include error handling
- Be 200+ lines for Buffer Overflow
- Say "# Generated by Mistral AI" at top
- Have implementation details specific to target

Template code will:
- Be much shorter
- Say "# This is a placeholder"
- Be generic without target specifics

## Debugging Steps

### Issue: Still seeing placeholder

**Check 1: Is AI provider initialized?**
```python
from enhanced_exploit_template import ExploitTemplate, AIProviderConfig
AIProviderConfig.setup_from_config()
template = ExploitTemplate(use_ai=True)
print(f"AI enabled: {template.use_ai}")
print(f"AI generator: {template.ai_gen}")
print(f"Provider: {template.ai_gen.provider if template.ai_gen else 'None'}")
```

**Check 2: Can AI be called directly?**
```python
from enhanced_exploit_template import AIExploitGenerator
gen = AIExploitGenerator()
result = gen.generate_exploit("Hello, write Python code")
print(result[:200])
```

**Check 3: Is config file valid?**
```bash
cat .hades_config.json
# Should show your API key, not empty
```

### Issue: "Template-generated" instead of "AI-generated"

**Cause:** AI call returned None, falling back to template

**Solution:**
1. Check API key is valid
2. Check internet connection
3. Check logs for error messages
4. Run VERIFY_AI_USAGE.py

## Forced AI Generation

The code now:

1. **Requires** AI providers to be available
   - Returns empty dict if no providers found
   - Forces fallback to prompts instead

2. **Validates** AI generator is initialized
   - Checks `template.ai_gen` is not None
   - Checks client is properly created

3. **Logs everything**
   - Provider name
   - Character count
   - AI vs template decision
   - Full error traces

4. **Prioritizes AI**
   - Always tries AI first
   - Only falls back if it fails
   - Logs when fallback happens

## Complete Workflow

```
HadesAI starts
  ↓
User opens Exploit Generator tab
  ↓
User loads .exe file
  ↓
User clicks "Analyze"
  ↓
User clicks "Generate Exploits"
  ↓
ExploitGeneratorWorker._try_enhanced_generation() runs
  ├─ Loads .hades_config.json
  ├─ Checks AI providers available
  ├─ Creates ExploitTemplate with use_ai=True
  ├─ Verifies template.ai_gen is not None
  ├─ For each payload type:
  │  ├─ Calls template.generate_exploit_payload()
  │  │  └─ Calls ai_gen.generate_exploit(prompt)
  │  │     └─ Calls Mistral API
  │  │        ├─ Logs provider and character count
  │  │        └─ Returns AI-generated code
  │  └─ Logs "AI-generated"
  ├─ Returns exploits dict
  └─ Falls back to prompts ONLY if enhanced fails
  ↓
UI displays exploit code
```
